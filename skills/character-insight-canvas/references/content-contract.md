# Content Contract

The explorer is character-first. Its order is a reading contract, not decoration.

## Layer 1: Character brief

The default drawer must answer the character in roughly 30–60 seconds.

Required blocks:

1. Exact name and source scope.
2. One-sentence thesis: public face + central contradiction + direction of change.
3. Short introduction in concrete social terms.
4. `第一眼 → 深入后` comparison.
5. Identity coordinates.
6. Three to five behavior-based traits.
7. Key relationships written as tensions.
8. Visual impression reference.
9. A clear action into the core path.

Bad trait: `她很坚强。`

Usable trait: `她在失控时会先行动、再解释，并愿意用关系破裂换回决定权。`

## Layer 2: Core path

Use 5–9 ordered steps. Each step contains:

- change, not merely an event;
- what pressure or relationship caused it;
- what choice the character made;
- the target view and evidence node.

The path summarizes the character arc. It must not duplicate every plot beat.

## Layer 3: Evidence views

Choose the smallest set of views that answers the material's distinct questions. One view is valid; more than four is also valid when every view earns its place. Never target a map count.

`人物关系` is normally required. Add other views only when they explain a real dimension:

- chronology / subjective experience;
- faction or institution;
- rules, abilities, or world mechanism;
- theme and recurring motif;
- adaptation or version comparison;
- public persona versus private behavior.

Every view needs its own question. If two views answer the same question, merge them.

## Layer 4: Node detail

Each node contains:

- concise title;
- summary;
- optional body for nuance;
- certainty label;
- one-hop relations;
- evidence links and source type.

The drawer stack grows outward:

```text
角色速览 → 核心路径 → 证据节点 → 关系节点
```

New layers cover old layers but leave a clickable rail. Returning truncates the trail; it does not create a second navigation history.

## Medium neutrality

Use `work`, `source scope`, `installment`, and `version` in the schema. Put medium-specific terms such as route, episode, chapter, season, volume, cut, or ending inside source-scope data.

Do not require game-only concepts. A character may come from animation, manga, novel, film, theatre, audio drama, mythology, or mixed adaptation material.
